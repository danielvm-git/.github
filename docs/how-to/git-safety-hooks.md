# Git safety hooks (`guard-git`)

bigpowers ships `guard-git` hooks that block dangerous git operations in AI agent sessions and enforce commit discipline:
- `git push` (to main/master)
- `git push --force`
- `git reset --hard`
- `git clean -fd`
- `git branch -D`
- `git checkout .` / `git restore .`
- Non-Conventional Commits messages
- **`Co-authored-by:` footers** (P1 — CONVENTIONS.md § Git Attribution)

## Installation

Run `bigpowers setup` in the repo — hooks are auto-linked for Claude Code, Cursor, Gemini CLI, and pi.

## Manual install (if `bigpowers setup` wasn't used)

```bash
# Claude Code
mkdir -p .claude/hooks
cp node_modules/bigpowers/scripts/hooks/pre-tool-use.sh .claude/hooks/

# Cursor
# .cursor/rules/guard-git.mdc is auto-generated by sync-skills.sh

# Gemini CLI
# .gemini/extensions/bigpowers/guard-git/ is auto-generated
```

## Verify hooks are active

```bash
# Should print hook path if installed
ls .claude/hooks/pre-tool-use.sh 2>/dev/null && echo "Claude Code: HOOKED" || echo "Claude Code: missing"
```

## What's blocked vs allowed

| Operation | Blocked? |
|-----------|----------|
| `git push origin main` | 🔴 Blocked |
| `git push origin feature-branch` | 🟢 Allowed |
| `git push --force` | 🔴 Blocked |
| `git reset --hard` | 🔴 Blocked |
| `git branch -D main` | 🔴 Blocked |
| `git branch -d feat/done` | 🟢 Allowed |
| `git checkout .` | 🔴 Blocked |
| Non-Conventional Commit message | 🔴 Blocked |
| `Co-authored-by:` in commit | 🔴 Blocked (P1) |
| `git commit -F file` with attribution | 🔴 Blocked (file content scanned) |

## Disabling (emergency only)

Set `GIT_BIGPOWERS_LAND=1` to bypass for intentional land operations (used by `scripts/land-branch.sh`).
